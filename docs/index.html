A data-grounded simulator for evaluating whether tool-using LLM agents can operate a supermarket across pricing, replenishment, supplier choice, inventory aging, reviews, external events, and cash-flow constraints.

What it measures

Operational policy formation under delayed feedback.

RetailBench is not a one-shot web task or a short tool-use test. The agent repeatedly chooses what to inspect, what to buy, how much to stock, when to change prices, and how to react when outcomes arrive days later.

The current public result package uses a survival-first selection rule: maximize operating days, then final net worth, then total sales. A privileged quality-based heuristic is shown as an oracle-style reference, not as a fair LLM-agent baseline.
